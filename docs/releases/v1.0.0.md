# SuperNoNo v1.0.0

SuperNoNo v1.0.0 is the first public release candidate for the desktop agent
companion.

## Highlights

- Compact Electron desktop pet UI with transparent always-on-top window.
- Minimal pet mode with bottom status bar and speech bubble.
- Tray menu for show/hide, task panel, settings, demo, and quit.
- Hover interaction for lightweight ambient pet motion.
- Local signal bridge on `127.0.0.1:4174`.
- Agent-neutral signal protocol for Codex, Claude Code, Cursor, generic CLI, and
  future adapters.
- Verified Codex Desktop plugin hooks:
  - `command_running` from `PreToolUse`
  - `step_done` from `PostToolUse`
  - `turn_ended` fallback from notify wrapper
- Codex adapter/plugin files included in the repository.

## Windows Portable Build

Download `SuperNoNo-win32-x64-v1.0.0.zip`, unzip it, and run:

```text
SuperNoNo.exe
```

This is a portable package. It does not install shortcuts or auto-updates.

The packaging script intentionally excludes generated local runtime files such as
`notify-wrapper.config.json` and `notify-observed.json`, so user-specific Codex
paths and local notify diagnostics are not included in the release archive.

## Codex Integration Quickstart

1. Start SuperNoNo and keep it running so the local bridge listens on
   `127.0.0.1:4174`.
2. Install the local Codex plugin by following
   `plugins/supernono-codex/INSTALL.md`.
3. Restart/open Codex Desktop and approve the `supernono-codex` hook trust prompt.
4. Run a real Codex tool action, for example a shell command such as
   `echo supernono-hook-test`.

Expected result: the pet reacts to `command_running` and `step_done` events with
`adapter: "codex-plugin-hooks"`. The notify wrapper may also send quiet
`turn_ended` fallback events with `adapter: "codex-desktop-notify"`.

## Assets and Privacy

- The v1.0 package includes the desktop pet visual assets used by this demo
  build. If you redistribute or commercialize a modified build, review the asset
  licenses or replace the character assets with your own.
- If Live2D assets fail to load, SuperNoNo falls back to its built-in SVG pet.
- Hook adapters send short, sanitized event summaries only. They do not send
  prompts, source text, full diffs, tokens, or secrets.
- Notify wrapper observation stores payload shape only, not conversation content.

## Known Notes

- Codex plugin hook setup still requires local configuration and hook trust.
- Windows `command_windows` in the Codex plugin currently uses a machine-specific
  Node path and should be productized in a future setup script.
- `legacy_notify` Windows error 206 is tracked as a separate fallback issue.
